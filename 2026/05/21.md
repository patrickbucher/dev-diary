# Thursday, 21 May 2026

I worked through chapter 7 in trpl, which is about modules, crates, etc. This was a good refresher. The Rust module system makes much more sense to me than the one used by Go or Python. The good thing about new programming languages is that they had the chance to learn from the shortcomings of the old ones. Rust is great in that respect. Older programming languages also get it right occasionally using proper tooling. However, Rust has that kind of tooling packaged with the language; unlike Java, which has a history of mediocre and annoying build systems.

Rust compilation is still somewhat slow, and so is Haskell's. Go is considerably quicker. But as soon as external dependencies are involved, all the environments regress to compilation times within the same magnitude. Caching makes this bearable.
